Image

Usually, when you want to hide and show some text, you add a ternary statement to a class. Hiding and showing text like this can easily be overused.

The code is still in the dom and is in the source code to be seen by users if inspected with dev tools. Also, if data points are missing or different arrays containing data need to be shown depending on the user's profile or something else dynamic, a simple thing becomes a overly complicated problem.

It's better to map over the data and use an object with a dynamic key set to a variable to choose which data to show; that way, you write less logic and clean up the code. You also don't have to do as much if a data point is missing as it just won't render.

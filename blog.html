A Ternary Operator May Not Be The Answer!

Usually, when you want to hide and show some text, you add a ternary operator to a class. Hiding and showing text like this is fine in some cases but can easily be overused.

The code is still in the dom when you hid things with classes and if inspected with dev tools it can be seen by users. This can be a bad thing depending on what is too be hidden and shown. Also, if data points are missing or different arrays containing data need to be shown depending on the user's profile or something else dynamic, more if else statments and operators need to be added for the logic to work which over complicates things.

It's better to map over the data and use an object with a dynamic key set to a variable to choose which data to show; that way, you write less logic and clean up the code. You also don't have to do as much if a data point is missing as it just won't render.
